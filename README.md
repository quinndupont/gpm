# Good Poetry Models (GPMs)

Teaching language models to revise — synthetic prompt training for poetry generation.

Language models produce poetry that is formally competent but stylistically empty—lacking the distinctive voice that makes verse matter. This project implements **synthetic education**: a methodology that trains models not on poems but on poetry instruction—critique, revision dialogue, and craft lessons—generated by a frontier model in a consistent pedagogical persona. A two-phase pipeline produces a structured corpus that fine-tunes separate **Educator** (mentor/critic) and **Poet** (generator) models, which operate in a multi-agent revision loop at inference. **RevFlux**, a process-oriented benchmark, measures revision dynamics rather than output quality. The trained system exhibits substantive critique-responsive revision, coarse-to-fine editing dynamics, and prompt-sensitive behaviour absent in the vanilla baseline—demonstrating that encoding pedagogical process rather than poetic product enables qualitatively different revision behaviour.

**Live:** These GPMs power [Bad Metaphor](https://badmetaphor.studio), including **Volta**—geometric aesthetics visualizations of poem structure and revision trajectories.

See [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) and [docs/DESIGN.md](docs/DESIGN.md).

## Training methods

| Method | Status | Notes |
|--------|--------|-------|
| **QLoRA** | ✓ | NF4 base, low-rank adapters (ΔW = B·A), loss on response tokens only. r=64, α=128; target attn + MLP. |
| **Educator SFT** | ✓ | (prompt → critique) pairs; prompt diversity critical; system prompt anchors persona. |
| **Poet filtered SFT** | ✓ | Deterministic rhyme algo → letter patterns → filter by compliance (≥0.6). "Write a {form} ({pattern})" → poem. |
| **Poet SRPO** | ✓ | Self-Refinement Policy Optimization: dual loss (40% generation + 60% revision). Trains poet to both generate and self-revise. Optional educator-scored rewards (imagery, originality, sentiment, form, voice). L = α·L_gen + (1-α)·L_rev + β·KL. |
| **Partial credit reward** | ✓ | Tiered scoring: perfect 1.0 → slant 0.6 → assonance 0.3 → none 0.0. Avoids forced rhyme at expense of meaning. |

Full technical summary: [docs/DESIGN.md](docs/DESIGN.md) (§ GPM Training Methods).

## Data prep

Add raw poetry to `data/raw/good/` and `data/raw/bad/`. Format: `.json` or `.jsonl` with `{author, title, poem}` per item; `.txt` as plain poem. Contrastive learning: bad poems paired with good for comparisons.

## Full pipeline

| Step | What | Frontier API | Local |
|------|------|--------------|-------|
| 1 | Hard tasks (critiques, comparisons, revision_briefs) | yes | — |
| 2 | Train interim educator on seed | 0 | Modal GPU |
| 3 | Export + download interim educator | 0 | Modal + local |
| 4 | Local educator generates briefs, autopsies, lessons | 0 | llama.cpp |
| 5 | Poet pairs | yes | — |
| 6 | Rhyme pairs + approval examples | yes | — |
| 7 | Prepare full training data (educator + poet with rhyme) | 0 | Combines JSONL |
| 8 | Train final educator + poet (SFT; optional SRPO Stage 2) | 0 | Modal GPU |
| 9 | Export + download final models | 0 | Modal + local |

Seed data (critiques, comparisons, revision_briefs, poet_pairs, rhyme_pairs) is generated by a **frontier model** in the educator persona. Briefs, autopsies, and lessons are then generated by the **interim educator** (trained on seed, run locally). The workflow also runs `generate_dialogues.py` and `generate_approval_examples.py` before the final prepare step. The poet is trained on general pairs + rhyme pairs + strong-rhyme poems.

**SRPO (Stage 2):** Optional refinement training for the poet. Generate trajectories with `generate_srpo_data_v2.py` (hybrid variance + model-gap). Uses Bedrock (Gemma, GLM, Claude) for draft_0 → critique → draft_1. Rewards: rhyme-only (default) or multi-dimensional educator scoring (`use_educator_scoring: true`) for imagery, originality, sentiment, form, voice. Trajectories filtered by improvement (`reward_1 - reward_0 >= 0.05`) and quality (`reward_1 >= 0.5`). The SRPO-trained poet learns both generation (40% loss weight) and self-revision (60% loss weight) in a single training run, enabling critique-responsive revision at inference without additional Educator calls.

### 1. Data generation (frontier API)

**Hard tasks:**
| Script | Input | Output |
|--------|-------|--------|
| `generate_critiques_seed` | raw good + bad | critiques_seed.jsonl |
| `generate_comparisons` | good + bad pairs | comparisons.jsonl |
| `generate_revision_briefs` | critiques_seed | revision_briefs_seed.jsonl |
| `generate_poet_pairs` | briefs + revision_briefs | pairs.jsonl |
| `generate_rhyme_pairs` | form-specific | rhyme poet + educator data |
| `generate_approval_examples` | — | educator approval/rejection examples |
| `generate_srpo_data_v2` (Stage 2) | poet prompts | SRPO trajectories (hybrid; optional educator scoring) |

**Easier (interim educator or frontier):**
| Script | Input | Output |
|--------|-------|--------|
| `generate_briefs` | raw good | briefs.jsonl |
| `generate_autopsies` | raw bad | autopsies.jsonl |
| `generate_lessons` | craft questions | lessons.jsonl |

```bash
# Full run (Stage 1: Educator + Poet SFT)
python scripts/data_generation/generate_critiques_seed.py --limit-good 200
python scripts/data_generation/generate_comparisons.py
python scripts/data_generation/generate_revision_briefs.py --limit 50
python scripts/data_generation/prepare_training_data.py --interim-educator --educator-only
# Upload, train interim educator, export, download; then:
python scripts/data_generation/generate_with_local_educator.py --all --limit-briefs 200 --limit-lessons 10
python scripts/data_generation/generate_poet_pairs.py
python scripts/data_generation/generate_dialogues.py
python scripts/data_generation/generate_rhyme_pairs.py --replace
python scripts/data_generation/generate_approval_examples.py --replace

# Optional: SRPO trajectories (Stage 2 - requires AWS Bedrock)
python scripts/data/generate_srpo_data_v2.py --config config/srpo_data_generation_v2.yaml --limit 5000
```

### 2. Prepare training data

```bash
python scripts/data_generation/prepare_training_data.py [--educator-only] [--poet-only] [--quality-gate] [--min-samples N] [--seed N]
```

Combines outputs into chat format. Poet data includes general pairs, rhyme pairs, and strong-rhyme poems (quality-gated at `strict_rhyme_density >= 0.6`). `--quality-gate`: run quality gate on educator data; fail if pass rate < 90%. `--min-samples N`: requires ≥N examples; if set, caps train/valid (quick test).

### 3. Upload (Modal or S3)

**Modal:** `python scripts/modal/upload_data.py` — uploads `data/educator_training/`, `data/poet_training/` to `poetry-data` volume.

**SageMaker:** `python scripts/sagemaker/upload_to_s3.py` — uploads `data/educator_training/`, `data/poet_training/`, and `data/srpo_training/` (if exists) to S3 (uses `config/sagemaker.yaml`: bucket, region). Validate: `python scripts/sagemaker/validate_setup.py`.

**Note:** For SRPO training, generate trajectories first with `scripts/data/generate_srpo_data_v2.py`, then run upload.

**Modal:**
```bash

# Orchestrate

```

**SageMaker:** Configure `config/sagemaker.yaml` (S3 bucket, IAM role, `instance_type`, `hf_secret_name`). Then:
```bash

# Stage 2 (optional): SRPO

python scripts/data/generate_srpo_data_v2.py --config config/srpo_data_generation_v2.yaml --limit 5000
# 2. Upload to S3 (includes trajectories_v2.jsonl):

python scripts/sagemaker/export_gguf_sagemaker.py  # merge + GGUF
```

## Poet training (two-stage)

**Stage 1 (SFT):** The unified poet is trained on general pairs + rhyme pairs + strong-rhyme poems (quality-gated via `strict_rhyme_density >= 0.6`). Data from `generate_poet_pairs.py`, `generate_rhyme_pairs.py`, and `select_strong_rhyme_poems.py`. Config: `config/poet_training.yaml`.

**Stage 2 (optional - SRPO):** Self-Refinement Policy Optimization. Trains the poet to both generate AND self-revise based on critique using dual-objective loss: 40% generation (draft_0) + 60% revision (draft_1 | critique). Trajectories (prompt → draft_0 → critique → draft_1) generated via `generate_srpo_data_v2.py` (hybrid variance + model-gap). Optional multi-dimensional educator scoring for rewards. Config: `config/srpo_training.yaml`. See `scripts/data/generate_srpo_data_v2.py` and [scripts/training/README.md](scripts/training/README.md) for details.

## Benchmarking / eval

### RevFlux (revision process benchmark)

`scripts/benchmarks/rev_flux/` measures **process dynamics**, not outcome quality: how much each line changes during revision, where change concentrates, and how this varies by prompt category and revision length. Only the trained model runs the revision loop; vanilla models produce initial drafts only.

```bash
python scripts/benchmarks/rev_flux/run_harness.py [--visualize]
python scripts/benchmarks/rev_flux/run_harness.py --models trained llama3.1-8b --limit 2
python scripts/benchmarks/rev_flux/run_harness.py --list-models
```

Models: `config/rev_flux_models.yaml`. See [scripts/benchmarks/rev_flux/README.md](scripts/benchmarks/rev_flux/README.md).

### Eval scripts

| Script | Purpose |
|--------|---------|
| `rhyme_analyzer.py` | CMU-based rhyme detection: scheme, strict vs slant. Input: poem text. |
| `meter_analyzer.py` | Scansion via CMU: metrical pattern, consistency. |
| `form_registry.py` | Form definitions (sonnet, villanelle, limerick, etc.). |
| `select_strong_rhyme_poems.py` | Strong CMU-verified rhyme → `data/annotated/strong_rhyme_poems.jsonl`. |
| `voice_consistency.py` | Educator outputs: banned phrases, specificity; input JSONL with `output`/`response`/`content`. |
